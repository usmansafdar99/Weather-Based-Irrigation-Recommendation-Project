## Weather-Based Irrigation Recommendation System

This project predicts crop water requirements using weather and soil data and provides actionable irrigation recommendations (High / Medium / Low) through a Streamlit dashboard.

### Main components

- **Data pipeline**: Ingestion, preprocessing, feature engineering, and dataset splitting.
- **Modeling**: Regression model to predict a numeric water requirement score derived from `Irrigation_Need` plus evaluation (MAE, RMSE, R²).
- **Irrigation logic**: Maps numeric water requirement into recommendations:
  - High → "Irrigate fully"
  - Medium → "Irrigate partially"
  - Low → "No irrigation today"
- **Dashboard**: Streamlit app to visualize predictions for new inputs and display recommendations.

### Dataset

Place your CSV dataset in `Dataset/irrigation_prediction.csv` (already provided).  
Target column in the provided dataset is `Irrigation_Need` (categorical: Low / Medium / High).  
The code creates a **numeric water requirement score** from this categorical label for regression.

### Setup

```bash
pip install -r requirements.txt
```

### Training the model

From the project root:

```bash
python train_model.py
```

This will:

- Load `Dataset/irrigation_prediction.csv`
- Preprocess and engineer features
- Train a regression model
- Evaluate it (MAE, RMSE, R²)
- Save the trained model and preprocessing pipeline to the `models/` directory

### Running the dashboard

```bash
streamlit run app.py
```

The dashboard will:

- Let you enter weather and soil conditions (and select crop, season, etc.)
- Predict daily and weekly water requirement scores
- Display the irrigation recommendation as color-coded text (green/yellow/red)

### Project structure

- `Dataset/irrigation_prediction.csv` – input dataset
- `train_model.py` – end-to-end training and evaluation script
- `app.py` – Streamlit dashboard
- `src/`
  - `config.py` – configuration constants (paths, thresholds, etc.)
  - `data.py` – data loading utilities
  - `preprocessing.py` – preprocessing and feature engineering
  - `modeling.py` – model creation, training, and persistence
  - `irrigation_logic.py` – irrigation recommendation logic
  - `metrics.py` – evaluation helpers
- `models/` – saved model and preprocessing artifacts
- `reports/figures/` – EDA plots generated by `eda.py`

### Notes and assumptions

- The original `Irrigation_Need` (Low / Medium / High) is used both:
  - As the **source** for a numeric water requirement score for regression.
  - As a **classification-like** label for human-readable recommendations.
- Soil moisture deficit is approximated as the difference between an assumed field capacity and the current `Soil_Moisture` value.
  - You can adjust the assumed field capacity in `src/config.py`.
